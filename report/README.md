# DTS304 Retail Management System

## Data Management I Project

# Project Description

The DTS304 Retail Management System is a data management project developed to simulate the design and implementation of an information management system for a retail business.

The project demonstrates:

- Data collection
- CSV data representation
- Relational database design
- Database indexing
- SQL retrieval
- Search algorithm implementation

The system manages customer purchase information and demonstrates efficient data storage and retrieval techniques.

# Project Objectives

The project aims to:

- Collect retail transaction data.
- Store information in structured format.
- Create a relational database.
- Implement database indexes.
- Retrieve information efficiently.
- Compare Sequential Search and Binary Search algorithms.

# Features

## Dataset Management

- Customer purchase records
- CSV storage
- Excel representation

## Database Features

- Relational database
- Primary index
- Secondary index
- Non-clustered index
- SQL queries

## Search Features

- Sequential Search
- Binary Search
- Performance comparison

# Installation

Clone or download the project.

Navigate into the project directory:
bash
cd DTS304_Retail_Management_System

Install Python requirements:
bash
pip install -r requirements.txt

# Database Setup

Open:

database/retail_database.sql

Execute the SQL script using:

- MySQL
- PostgreSQL
- XAMPP phpMyAdmin

The script will:

- Create database
- Create table
- Insert records
- Create indexes
- Execute retrieval queries

# Running Search Simulation

Navigate to:

python/

Run:
bash
python search_simulation.py

The program will:

1. Load the CSV dataset.
2. Perform Sequential Search.
3. Perform Binary Search.
4. Compare execution time.
